# دليل الاختبار السريع لنظام تسجيل الدخول

## 🚀 البدء السريع

### الخطوة 1: تشغيل الخادم
```bash
npm run dev
```

### الخطوة 2: فتح صفحة تسجيل الدخول
افتح المتصفح على: `http://localhost:3000/login`

### الخطوة 3: إنشاء/استخدام حساب تجريبي

#### الخيار 1: استخدام زر الحساب التجريبي (في بيئة التطوير)
- في صفحة تسجيل الدخول، ستجد زر "تسجيل الدخول بحساب تجريبي"
- اضغط عليه وسيتم إنشاء الحساب تلقائياً وتسجيل الدخول

#### الخيار 2: إنشاء الحساب يدوياً
```bash
# تشغيل سكريبت إنشاء الحساب التجريبي
npx tsx scripts/create-test-account.ts
```

#### الخيار 3: استخدام API مباشرة
```bash
curl -X POST http://localhost:3000/api/auth/test-account
```

### بيانات الحساب التجريبي:
- **البريد الإلكتروني:** `test@example.com`
- **كلمة المرور:** `Test123!@#`

## ✅ قائمة الاختبار السريعة

### اختبار 1: تسجيل الدخول الناجح
1. [ ] افتح صفحة تسجيل الدخول
2. [ ] أدخل البريد: `test@example.com`
3. [ ] أدخل كلمة المرور: `Test123!@#`
4. [ ] اضغط "تسجيل الدخول"
5. [ ] **النتيجة المتوقعة:** يجب أن يتم تسجيل الدخول بنجاح وإعادة التوجيه

### اختبار 2: التحقق من صحة البيانات
1. [ ] أدخل بريد إلكتروني غير صحيح (مثل: `test`)
2. [ ] **النتيجة المتوقعة:** يجب أن يظهر خطأ "البريد الإلكتروني غير صالح"

### اختبار 3: كلمة مرور خاطئة
1. [ ] أدخل بريد صحيح: `test@example.com`
2. [ ] أدخل كلمة مرور خاطئة: `WrongPassword`
3. [ ] **النتيجة المتوقعة:** يجب أن يظهر خطأ "بيانات تسجيل الدخول غير صحيحة"

### اختبار 4: Rate Limiting
1. [ ] حاول تسجيل الدخول 5 مرات ببيانات خاطئة
2. [ ] **النتيجة المتوقعة:** يجب أن يظهر رسالة "تم تعليق محاولات تسجيل الدخول مؤقتاً"

### اختبار 5: CAPTCHA
1. [ ] بعد 3 محاولات فاشلة
2. [ ] **النتيجة المتوقعة:** يجب أن يظهر CAPTCHA قبل إمكانية المحاولة مرة أخرى

### اختبار 6: حفظ الحالة
1. [ ] سجل الدخول بنجاح
2. [ ] افتح Developer Tools → Application → Local Storage
3. [ ] **النتيجة المتوقعة:** يجب أن تجد `auth_token` و `user_data`

### اختبار 7: Refresh Token
1. [ ] سجل الدخول بنجاح
2. [ ] انتظر دقيقة
3. [ ] افتح Developer Tools → Network
4. [ ] **النتيجة المتوقعة:** يجب أن يتم تحديث الرمز المميز تلقائياً

## 🔍 فحص Console و Network

### في Console (F12 → Console):
- ✅ لا توجد أخطاء JavaScript
- ✅ رسائل التحذير واضحة
- ✅ Token يتم حفظه بشكل صحيح

### في Network (F12 → Network):
- ✅ طلبات API تعمل بشكل صحيح
- ✅ استجابة `/api/auth/login` تحتوي على `token` و `user`
- ✅ Status Code: 200 للنجاح، 401 للفشل

## 🐛 الأخطاء الشائعة وحلولها

### خطأ: "Cannot connect to server"
- **الحل:** تأكد من تشغيل الخادم بـ `npm run dev`

### خطأ: "User not found"
- **الحل:** أنشئ الحساب التجريبي أولاً:
  ```bash
  npx tsx scripts/create-test-account.ts
  ```

### خطأ: "Invalid token format"
- **الحل:** امسح localStorage و sessionStorage وأعد المحاولة

### خطأ: "Rate limited"
- **الحل:** انتظر 15 دقيقة أو أعد تشغيل Redis

## 📊 اختبارات إضافية

### اختبار API مباشرة:

#### تسجيل الدخول الناجح:
```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "Test123!@#",
    "rememberMe": false
  }'
```

#### تسجيل الدخول ببيانات خاطئة:
```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "wrong@example.com",
    "password": "WrongPassword",
    "rememberMe": false
  }'
```

### التحقق من Token:
1. سجل الدخول بنجاح
2. افتح Console
3. اكتب: `localStorage.getItem('auth_token')`
4. **النتيجة المتوقعة:** يجب أن ترى الرمز المميز

# ✅ قائمة التحقق النهائية - تسجيل الدخول عبر Google

## 📋 ملف `.env.local` - المطلوب

يجب أن يحتوي الملف على **3 متغيرات** فقط:

```env
GOOGLE_CLIENT_ID=586804623377-fcf7nrh1g02hd4fe6urreek6pi5rtk8l<GOOGLE_CLIENT_ID>
GOOGLE_CLIENT_SECRET=<GOOGLE_CLIENT_SECRET>4V-cqt0Eh2H1h5V0Tubo-5mo9S_4
NEXT_PUBLIC_BASE_URL=http://localhost:3000
```

### ✅ التحقق:
- [ ] `GOOGLE_CLIENT_ID` موجود وليس فارغاً
- [ ] `GOOGLE_CLIENT_SECRET` موجود وليس فارغاً
- [ ] `NEXT_PUBLIC_BASE_URL=http://localhost:3000` موجود
- [ ] لا توجد مسافات حول علامة `=`
- [ ] لا توجد علامات اقتباس حول القيم

---

## 🔧 إعدادات Google Cloud Console - المطلوبة

### 1. OAuth 2.0 Client ID
- [ ] Client ID موجود
- [ ] Client Secret موجود
- [ ] **Authorized redirect URIs** يحتوي على:
  ```
  http://localhost:3000/api/auth/google/callback
  ```
  ⚠️ **يجب أن يطابق بالضبط** (حرف بحرف)

### 2. OAuth Consent Screen
- [ ] User type: **External**
- [ ] App name: مكتمل
- [ ] User support email: مكتمل
- [ ] Developer contact information: مكتمل
- [ ] **Scopes** مضافة:
  - [ ] `.../auth/userinfo.email` أو `https://www.googleapis.com/auth/userinfo.email`
  - [ ] `.../auth/userinfo.profile` أو `https://www.googleapis.com/auth/userinfo.profile`
  - [ ] `openid` أو `https://www.googleapis.com/auth/openid`
- [ ] إذا كان في وضع **Testing**:
  - [ ] بريدك الإلكتروني موجود في **Test users**

## 🚀 خطوات التشغيل

### 1. إعادة تشغيل الخادم
```bash
# أوقف الخادم (Ctrl+C) إذا كان يعمل
npm run dev
```

⚠️ **مهم جداً:** يجب إعادة تشغيل الخادم بعد أي تغييرات في `.env.local`

### 2. التحقق من أن الزر يظهر
- [ ] افتح: `http://localhost:3000/login`
- [ ] يجب أن ترى زر **"تسجيل الدخول بجوجل"**
- [ ] إذا لم يظهر، تحقق من:
  - ملف `.env.local` موجود وصحيح
  - الخادم تم إعادة تشغيله
  - لا توجد أخطاء في console

### 3. اختبار تسجيل الدخول
- [ ] انقر على زر "تسجيل الدخول بجوجل"
- [ ] يجب أن تفتح نافذة Google
- [ ] اختر حساب Google
- [ ] امنح الصلاحيات المطلوبة
- [ ] يجب أن يتم تسجيل الدخول تلقائياً

## ❌ المشاكل الشائعة

### المشكلة: زر Google لا يظهر
**الحل:**
1. تحقق من ملف `.env.local`
2. أعد تشغيل الخادم
3. تحقق من console logs

### المشكلة: خطأ "Access blocked: Tolo's request is invalid"
**الحل:**
1. تحقق من Redirect URI في Google Console
2. تحقق من OAuth Consent Screen
3. إذا كان Testing، أضف بريدك في Test users

### المشكلة: خطأ "redirect_uri_mismatch"
**الحل:**
1. تأكد من أن Redirect URI في Google Console هو بالضبط:
   ```
   http://localhost:3000/api/auth/google/callback
   ```
2. تأكد من أن `NEXT_PUBLIC_BASE_URL` في `.env.local` هو:
   ```
   http://localhost:3000
   ```
3. تأكد من عدم وجود مسافات أو أحرف إضافية

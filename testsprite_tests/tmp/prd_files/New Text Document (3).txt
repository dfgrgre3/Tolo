أنت مدير منتج تقني أول (Senior Technical Product Manager) ومهندس برمجيات (Software Architect).

المهمة: قم بتحليل قاعدة البيانات البرمجية المرفقة لمشروع "Tolo" وقم بكتابة وثيقة مواصفات المنتج (Product Requirement Document - PRD) شاملة واحترافية. يجب أن تعكس الوثيقة الوظائف الموجودة فعليًا في الكود والهيكل التقني للمشروع.

هيكل الوثيقة المطلوبة:

ملخص تنفيذي (Executive Summary):

ما هو المنتج؟ (منصة تعليمية ذكية).

من هم المستخدمون المستهدفون؟ (طلاب، معلمين، مشرفين).

ما هي القيمة الأساسية (Core Value Proposition)؟ (الدمج بين التعليم، الذكاء الاصطناعي، والتلعيب).

الميزات الوظيفية التفصيلية (Functional Requirements):

يجب تحليل المجلدات التالية بدقة:

نظام المصادقة والأمان (auth, security): اشرح بالتفصيل دعم المصادقة البيومترية (Biometric)، المصادقة الثنائية (2FA/TOTP)، رموز الاسترداد، تتبع الجلسات، وسجلات الأمان.

النظام التعليمي (courses, exams, lessons): كيف يتم عرض الدروس؟ كيف يعمل نظام الامتحانات وتوليد الأسئلة؟ إدارة الموارد والمكتبة.

الذكاء الاصطناعي (ai): اشرح وظائف مساعد الذكاء الاصطناعي (AI Assistant)، مولد الامتحانات (Exam Generator)، والبحث عن المعلمين.

التلعيب والمكافآت (gamification): تفاصيل نظام الإنجازات (Achievements)، لوحة المتصدرين (Leaderboard)، والنقاط.

التواصل والمجتمع (chat, forum, social): نظام الدردشة الفوري (WebSocket)، المنتدى، والإشعارات.

إدارة الوقت والمهام (time, schedule, tasks): الجدول الدراسي، تتبع وقت المذاكرة، والتذكيرات.

المتطلبات غير الوظيفية (Non-Functional Requirements):

الأداء (performance): استراتيجيات التخزين المؤقت (Redis/Caching)، وتحسين التحميل.

البنية التحتية (k8s, terraform): اشرح استخدام Kubernetes، Docker، واستراتيجيات النشر (CI/CD).

المراقبة (monitoring): أدوات مثل Prometheus، Grafana، و ELK Stack.

تصميم البيانات (Data Model):

بناءً على ملفات prisma/schema.prisma، قدم ملخصًا للعلاقات الرئيسية (User, Course, Exam, Session, Gamification).

واجهة المستخدم والتجربة (UI/UX):

تحليل المكونات (components/ui)، دعم اللغات (RTL/Arabic)، والوضع الليلي (Theme Toggle).

ملاحظة: اعتمد في تحليلك على أسماء الملفات، محتوى ملفات الـ Markdown التوثيقية (مثل ADVANCED_AUTH_SYSTEM.md)، وهيكل مجلد src.

ملخص سريع لمواصفات المنتج (بناءً على نظرتي الأولية للكود):
إذا كنت تريد مني البدء فورًا بكتابة المواصفات بناءً على ما أراه الآن، فهذا ملخص مبدئي لما يحتويه مشروعك:

اسم المنتج: Tolo (منصة تعليمية ذكية).

1. الميزات الأساسية (Core Features):

نظام أمان متقدم جدًا: يتجاوز تسجيل الدخول التقليدي ليشمل تسجيل الدخول بالبصمة (WebAuthn)، التحقق بخطوتين (TOTP)، روابط سحرية (Magic Links)، وكشف التسلل (Intrusion Detection).

الذكاء الاصطناعي (AI):

توليد امتحانات تلقائيًا.

مساعد دراسي (Chatbot) للطلاب.

بحث ذكي عن المعلمين وتوصيات بالمحتوى.

نظام إدارة التعلم (LMS):

دعم للكورسات، الدروس، والملفات المرفقة.

نظام امتحانات متكامل مع تصحيح تلقائي ورصد درجات.

التلعيب (Gamification):

نظام مكافآت لتحفيز الطلاب (Badges, Points).

لوحات شرف ومنافسات.

إدارة الوقت:

جدول دراسي أسبوعي (Weekly Schedule).

مؤقت للمذاكرة (Pomodoro/Time Tracker).

2. التقنيات المستخدمة (Tech Stack):

Frontend: Next.js (App Router), React, Tailwind CSS, TypeScript.

Backend: Next.js API Routes, GraphQL (TypeGraphQL).

Database: Prisma ORM (مع قاعدة بيانات SQL).

Real-time: WebSockets (للدردشة والإشعارات الفورية).

Infrastructure: Kubernetes (لإدارة الحاويات)، Terraform (للبنية التحتية ككود)، Redis (للكاش)، ELK Stack (للسجلات).

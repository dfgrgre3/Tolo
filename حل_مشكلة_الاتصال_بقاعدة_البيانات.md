# حل مشكلة الاتصال بقاعدة البيانات

## 📋 نظرة عامة

تم إصلاح وتحسين نظام الاتصال بقاعدة البيانات مع إضافة الميزات التالية:

1. ✅ إعادة المحاولة التلقائية عند فشل الاتصال
2. ✅ معالجة محسنة للأخطاء (EPERM, P1001, P1017)
3. ✅ التحقق التلقائي من صحة الاتصال
4. ✅ إعادة الاتصال التلقائية عند انقطاع الاتصال
5. ✅ أدوات تشخيص واختبار الاتصال

## 🔧 الملفات التي تم تحسينها

### 1. `src/lib/db.ts`
- إضافة دالة `ensureDatabaseConnection()` لضمان الاتصال
- تحسين دالة `checkDatabaseHealth()` مع إعادة المحاولة
- إضافة دالة `retryDatabaseOperation()` لإعادة المحاولة التلقائية

### 2. `src/lib/db-unified.ts`
- معالجة أخطاء EPERM (مشاكل الصلاحيات في Windows)
- إعادة الاتصال التلقائية عند انقطاع الاتصال
- معالجة أفضل لأخطاء Prisma

### 3. `src/lib/db-connection-helper.ts` (جديد)
- `diagnoseDatabaseConnection()` - تشخيص مشاكل الاتصال
- `testDatabaseConnection()` - اختبار الاتصال مع إعادة المحاولة
- `getDatabaseConnectionStatus()` - الحصول على حالة الاتصال
- `initializeDatabaseConnection()` - تهيئة الاتصال عند بدء التطبيق

### 4. `src/lib/db-service.ts`
- تحسين `executeQuery()` مع إعادة المحاولة التلقائية
- التحقق من الاتصال قبل تنفيذ الاستعلامات

### 5. `src/app/api/db/connection/route.ts` (جديد)
- API endpoint لاختبار وتشخيص الاتصال
- يمكن الوصول إليه عبر `/api/db/connection`

## 🚀 كيفية الاستخدام

### 1. التحقق من حالة الاتصال

```typescript
import { checkDatabaseHealth, ensureDatabaseConnection } from '@/lib/db';

// التحقق من صحة الاتصال
const isHealthy = await checkDatabaseHealth();
if (!isHealthy) {
  console.error('Database connection is not healthy');
}

// ضمان الاتصال (مع إعادة المحاولة)
const isConnected = await ensureDatabaseConnection();
```

### 2. تشخيص مشاكل الاتصال

```typescript
import { diagnoseDatabaseConnection } from '@/lib/db-connection-helper';

const diagnostics = await diagnoseDatabaseConnection();
console.log('Connection status:', diagnostics.isConnected);
console.log('Recommendations:', diagnostics.recommendations);
```

### 3. استخدام API للاختبار

```bash
# الحصول على حالة الاتصال
curl http://localhost:3000/api/db/connection

# تشخيص كامل
curl http://localhost:3000/api/db/connection?action=diagnose

# اختبار الاتصال
curl http://localhost:3000/api/db/connection?action=test

# تهيئة الاتصال
curl http://localhost:3000/api/db/connection?action=initialize
```

### 4. استخدام executeQuery مع إعادة المحاولة

```typescript
import { executeQuery } from '@/lib/db-service';

const user = await executeQuery(
  () => prisma.user.findUnique({ where: { id: userId } }),
  'Failed to find user',
  true, // إعادة المحاولة
  3 // عدد المحاولات
);
```

## 🔍 حل المشاكل الشائعة

### المشكلة 1: خطأ EPERM (Windows)

**السبب:** ملفات Prisma Client قيد الاستخدام من قبل عملية أخرى

**الحل:**
1. إغلاق جميع خوادم التطوير قيد التشغيل
2. إغلاق VS Code أو أي IDE آخر
3. تشغيل الأمر:
   ```bash
   npx prisma generate
   ```
4. إذا استمرت المشكلة، أعد تشغيل Terminal/IDE

### المشكلة 2: P1001 - Cannot reach database server

**السبب:** قاعدة البيانات غير قيد التشغيل أو غير متاحة

**الحل:**
1. التحقق من أن قاعدة البيانات تعمل
2. التحقق من `DATABASE_URL` في ملف `.env.local`
3. التحقق من إعدادات الجدار الناري والشبكة

### المشكلة 3: P1017 - Server has closed the connection

**السبب:** انقطع الاتصال بقاعدة البيانات

**الحل:**
- النظام الآن يعيد الاتصال تلقائياً
- إذا استمرت المشكلة، تحقق من إعدادات قاعدة البيانات

### المشكلة 4: Prisma Client not generated

**السبب:** Prisma Client لم يتم توليده بعد

**الحل:**
```bash
npx prisma generate
# أو
npm run prisma:generate
```

## ⚙️ متغيرات البيئة المطلوبة

تأكد من وجود المتغيرات التالية في `.env.local`:

```env
# SQLite (التطوير)
DATABASE_URL="file:./prisma/dev.db"

# أو PostgreSQL (الإنتاج)
DATABASE_URL="postgresql://user:password@localhost:5432/thanawy"

# إعدادات إضافية (اختيارية)
DB_CONNECTION_LIMIT=10
DB_MIN_POOL_SIZE=2
DB_MAX_POOL_SIZE=20
DB_CONNECTION_TIMEOUT=5000
DB_IDLE_TIMEOUT=30000
DB_RETRY_ATTEMPTS=3
DB_RETRY_DELAY=1000
```

## 📊 مراقبة الاتصال

النظام يقوم تلقائياً بمراقبة الاتصال في وضع التطوير. يمكنك تفعيل المراقبة في الإنتاج بإضافة:

```env
ENABLE_DB_STATS=true
```

## 🔄 آلية إعادة المحاولة

- **عدد المحاولات:** 3 محاولات افتراضياً
- **التأخير:** Exponential backoff (1s, 2s, 4s)
- **أنواع الأخطاء:** يتم إعادة المحاولة فقط على أخطاء الاتصال

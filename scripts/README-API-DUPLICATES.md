# سكربت فحص ازدواجية ملفات API Routes

## الوصف

هذا السكربت يفحص مشروع Next.js لاكتشاف ازدواجية ملفات API Routes والمشاكل المحتملة في بنية المسارات.

## المشاكل التي يكتشفها

1. **الملفات المكررة**: ملفات `route.ts` متعددة في نفس المجلد
2. **الملفات المشبوهة**: ملفات بأسماء تحتوي على أنماط مثل:
   - `-improved.ts`
   - `-enhanced.ts`
   - `-new.ts`
   - `-old.ts`
   - `-backup.ts`
   - `-copy.ts`
   - `-v2.ts`
3. **التضاربات المحتملة**: مسارات متشابهة قد تشير إلى ازدواجية

## الاستخدام

```bash
npm run check:api-duplicates
```

أو:

```bash
npm run check:api-routes
```

## المخرجات

السكربت يطبع تقريراً شاملاً يتضمن:

- **الملخص**: عدد الملفات المكررة، التضاربات، والملفات المشبوهة
- **الملفات المكررة**: قائمة بجميع الملفات المكررة مع مواقعها
- **الملفات المشبوهة**: قائمة بالملفات التي قد تكون نسخاً احتياطية أو نسخاً محسنة
- **التضاربات المحتملة**: مسارات متشابهة قد تحتاج إلى مراجعة

## أمثلة على المشاكل المكتشفة

### مثال 1: ملفات مكررة في نفس المجلد
```
src/app/api/reminders/
  ├── route.ts
  └── route-improved.ts  ❌ ملف مكرر
```

### مثال 2: ملفات مشبوهة
```
src/app/api/users/register/
  ├── route.ts
  └── route-enhanced.ts  ❌ ملف مشبوه
```

### مثال 3: تضارب في المسارات
```
/api/auth/session      ✅ مسار صحيح
/api/auth/sessions     ⚠️  قد يكون مكرراً أو مساراً مختلفاً
```

## التوصيات

عند اكتشاف مشاكل:

1. **احذف الملفات المكررة** أو ادمجها في ملف واحد
2. **تأكد من وجود ملف `route.ts` واحد فقط** في كل مجلد
3. **أعد تسمية الملفات المشبوهة** أو احذفها إذا لم تعد مستخدمة
4. **راجع التضاربات المحتملة** وأزل المسارات المكررة

## ملاحظات مهمة

- السكربت يتحقق فقط من ملفات `route.ts` و `route.tsx` في مجلد `src/app/api`
- المسارات المتداخلة في Next.js مسموحة (مثل `/api/auth` و `/api/auth/login`)
- بعض "التضاربات" المكتشفة قد تكون مسارات صحيحة - راجعها يدوياً

## كود الخروج

- `0`: لا توجد مشاكل
- `1`: تم اكتشاف مشاكل تحتاج إلى معالجة

## التكامل مع CI/CD

يمكن استخدام السكربت في CI/CD pipeline:

```yaml
# مثال GitHub Actions
- name: Check API Routes Duplicates
  run: npm run check:api-duplicates
```

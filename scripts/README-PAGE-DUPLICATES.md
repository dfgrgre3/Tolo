# سكربت فحص ازدواجية ملفات الصفحات (Pages)

## الوصف

هذا السكربت يفحص مشروع Next.js لاكتشاف ازدواجية ملفات الصفحات (Pages) في App Router والمشاكل المحتملة.

## المشكلة

في Next.js App Router، يتم تحميل فقط الملفات القياسية التالية كمسارات (Routes):
- `page.tsx` أو `page.ts`
- `layout.tsx` أو `layout.ts`

الملفات الأخرى مثل:
- `page-enhanced.tsx`
- `page-new.tsx`
- `page-advanced.tsx`
- `layout-new.tsx`

**لن يتم تحميلها كمسارات** وتعتبر ملفات مكررة تزيد حجم المشروع وتشتت المطورين.

## المشاكل التي يكتشفها

1. **الملفات المكررة**: ملفات `page-*.tsx` أو `layout-*.tsx` بجانب الملفات القياسية
2. **الملفات المشبوهة**: ملفات بأسماء تحتوي على أنماط مثل:
   - `page-enhanced.tsx`
   - `page-new.tsx`
   - `page-advanced.tsx`
   - `layout-new.tsx`
   - `page-improved.tsx`
   - `page-old.tsx`
   - `page-backup.tsx`
   - وغيرها...

## الاستخدام

### فحص الملفات المكررة

```bash
npm run check:page-duplicates
```

أو:

```bash
npm run check:pages
```

### تنظيف الملفات المكررة

**⚠️ تحذير: هذا الأمر سيحذف الملفات المكررة!**

```bash
npm run clean:page-duplicates
```

### فحص بدون حذف (Dry Run)

لرؤية ما سيتم حذفه دون حذف فعلي:

```bash
npm run clean:page-duplicates:dry
```

## المخرجات

السكربت يطبع تقريراً شاملاً يتضمن:

- **الملخص**: عدد الملفات المكررة والملفات المشبوهة
- **الملفات المكررة**: قائمة بجميع الملفات المكررة مع مواقعها
- **الملفات المشبوهة**: قائمة بالملفات التي قد تكون نسخاً احتياطية أو نسخاً محسنة

## أمثلة على المشاكل المكتشفة

### مثال 1: ملفات page مكررة

```
src/app/
  ├── page.tsx          ✅ الملف القياسي (يتم تحميله)
  └── page-enhanced.tsx ❌ ملف مكرر (لن يتم تحميله)
```

### مثال 2: ملفات layout مكررة

```
src/app/
  ├── layout.tsx        ✅ الملف القياسي (يتم تحميله)
  └── layout-new.tsx   ❌ ملف مكرر (لن يتم تحميله)
```

### مثال 3: ملفات profile مكررة

```
src/app/profile/
  ├── page.tsx          ✅ الملف القياسي (يتم تحميله)
  ├── page-new.tsx      ❌ ملف مكرر (لن يتم تحميله)
  └── page-advanced.tsx ❌ ملف مكرر (لن يتم تحميله)
```

## التوصيات

عند اكتشاف مشاكل:

1. **احذف الملفات المكررة**: Next.js يستخدم فقط الملفات القياسية
2. **انقل المحتوى المهم**: إذا كان لديك محتوى مهم في الملفات المكررة، انقله إلى الملف القياسي
3. **استخدم Git**: تأكد من أن التغييرات محفوظة في Git قبل الحذف
4. **راجع الكود**: تحقق من عدم وجود استيرادات (imports) للملفات المكررة

## كيفية التعامل مع الملفات المكررة

### إذا كان الملف المكرر يحتوي على محتوى مهم:

1. افتح الملف المكرر (مثل `page-enhanced.tsx`)
2. انسخ المحتوى المهم
3. الصقه في الملف القياسي (`page.tsx`)
4. احذف الملف المكرر

### إذا كان الملف المكرر غير مستخدم:

1. احذفه مباشرة باستخدام:
   ```bash
   npm run clean:page-duplicates
   ```

## منع المشكلة في المستقبل

1. **استخدم أسماء واضحة**: تجنب استخدام `-new`, `-enhanced`, `-advanced` في أسماء الملفات
2. **استخدم Git branches**: للتجارب والتطوير، استخدم فروع Git بدلاً من نسخ الملفات
3. **راجع الكود بانتظام**: شغّل `npm run check:page-duplicates` بانتظام
4. **استخدم pre-commit hooks**: يمكن إضافة السكربت إلى pre-commit hooks لمنع الالتزام بالملفات المكررة

## السكربتات المتاحة

- `check-page-duplicates.ts`: فحص الملفات المكررة
- `clean-page-duplicates.ts`: تنظيف وحذف الملفات المكررة

## ملاحظات

- السكربت يفحص فقط مجلد `src/app`
- الملفات في `node_modules` و `.next` يتم تخطيها تلقائياً
- السكربت لا يحذف الملفات القياسية (`page.tsx`, `layout.tsx`)
